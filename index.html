Info

I created a snake game where the user moves around the snake using the arrow keys collecting food.

As the snake eats each food, the snake gets bigger and faster, gradually making the game more difficult.

The game ends if the snake's head touches any part of its body, or if the snake hits the end of the game screen.

Features

The snake game comes complete with a full levelling and scoring system, where the game gets more difficult as the levels progress.

The UI also grants several options to the user for how they would like to play the game, such as turning the music or sound off, and whether they would like to play the game in light mode or dark mode.

With the sound setting on, the game features nice background music along with audio cues when the user reaches a new level or every time the snake eats the food.

There is also a pause game function if the user decides to take a break and also a new game function if the user wants to restart.

Once the game is over, the game displays the final score to the user, and allow the user to start a new game.

Design

The snake game features various shades of green which are consistent through each menu, and the colour changes to a darker shade if the user selects dark mode.

The game screen is placed at the center of the screen, and makes up nearly all of the webpage. The game screen is differentiated from the background with a darker shade, keeping consistent with the design colours.

The navbar is situated at the top of the webpage to allow the user to see all of the various information and settings easily on the screen whilst playing the game.

I created a weather app which allows the user to input an address by searching either a city/town name or postcode.

After searching the inputted location, the app then displays the current weather along with some other various useful information such as the current time, the day of the week and the temperature.

Once a location is entered, a navigation bar appears at the top of the screen along with some information and another search bar, where the user is also able to search a different location.

Features

The weather app features a dynamic background where the image changes depending on the weather of the location and the time of the location.

The UI also automatically changes to light mode or dark mode depending on the time of the inputted location.

The icons used to display the weather allow the user to quickly and easily see what the weather is like in the inputted location.

Design

The landing page of the weather app is simplistic and the user is greeted with a search bar which alerts the user to enter a location.

The app gets the data from a weather API I used, and displays the information in a clean, readable format to the user.

The location information is displayed on the navbar at the top of the screen and the main weather information is displayed in the center. Other useful/advanced weather information is displayed in a marquee under the navigation bar.

I created a dice game using JavaScript where the user can either play alone or with a friend.

There are different rules to the game depending on how many players are playing. However the rules are shown to the user(s) before starting the game.

Features

The user(s) can input their names at the beginning of the game to make it more personal.

Audio cues when starting a game, winning or losing further adds to the game experience.

When a player wins or loses, a message is displayed to the user(s) containing the final score, and whether they want to go to the main menu or start again.

Design

The dice game is structured as a single game screen which is displayed in the center of the page.

The text explaining the rules of the game is shown clearly to the user(s) and makes it easily understandable.

The buttons are nice and clear and the colour theme of the website is consistent on each game screen section.
